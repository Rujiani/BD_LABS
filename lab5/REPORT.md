# Отчётные материалы: 5 SQL-запросов (WikiSneaks)

## Предусловия

```bash
export PGDATABASE=wikisneaks
./Scripts/recreate_db.sh
python3 Scripts/ws_tools.py test-data --out db/test_data.sql --users 80 --documents 40 --versions-max 5 --seed 7
psql -v ON_ERROR_STOP=1 -f db/test_data.sql
```

Генератор `Scripts/ws_tools.py test-data` дополняет сид данными для отчётов:

- **Запрос 1** — `ws_login_history` с общими IP (2–4 пользователя на IP, несколько входов).
- **Запрос 3** — активные `ws_safe_ip` без входа > 1 месяца или без входов с этого IP; свежие входы с других IP; в seed/test_data есть заблокированные (`blocked_user` + несколько из test-data).
- **Запрос 4** — просмотры на прошлой полной календарной неделе (фильтр пользователей и матрица активности), даты через `CURRENT_DATE`.

## Соответствие обязательным требованиям

| Требование | Запрос |
|------------|--------|
| CTE во всех запросах | 1–5 |
| Не менее 7 строк результата | проверено на тестовых данных |
| JOIN 3+ таблиц, фильтры, **HAVING** | **1** (`shared_ip`) |
| **LEFT JOIN + IS NULL** (отсутствующие входы) | **3** |
| Оконная функция **ROWS BETWEEN** | при необходимости курса — не в итоговом SELECT Q5 (убрано как неиспользуемое) |
| Оконная функция смещения **LAG / LEAD** | при необходимости ТЗ — в другом запросе; в Q1 убран неиспользуемый LEAD |
| Pivot / crosstab (56 слотов + имя) | **4** |

## Запуск

```bash
psql -d wikisneaks -f db/queries/query01_shared_ip_pairs.sql
psql -d wikisneaks -f db/queries/query02_documents_report.sql
psql -d wikisneaks -f db/queries/query03_stale_safe_ip.sql
psql -d wikisneaks -f db/queries/query04_activity_pivot.sql

```

Пример проверки числа строк:

```bash
psql -d wikisneaks -A -t -f db/queries/query01_shared_ip_pairs.sql | wc -l
```

## Запрос 1 — пары пользователей с общим IP

**Файл:** `query01_shared_ip_pairs.sql`

**Логика.** `shared_ip` → `per_user` → `pairs` → `closest` (DISTINCT ON для ближайших по времени входов). См. комментарии в `query01_shared_ip_pairs.sql`.

**Пример (фрагмент):**

```
 IP-адрес      | Данные первого пользователя | ... | Разница, минуты
---------------+-----------------------------+-----+-----------------
 185.61.249.47 | td_7_0047_...               | ... |          149490.00
```

## Запрос 2 — отчёт по документам

**Файл:** `query02_documents_report.sql`

**Логика.** CTE `docs` (JOIN 3 таблиц; документы без версий не попадают из‑за INNER JOIN), `view_stats` и `fav_stats` — по одному проходу по просмотрам и избранному.

**Пример (фрагмент):**

```
 Название документа  | Общее число версий | Зрителей последней версии | Просмотров всех версий
---------------------+--------------------+----------------------------+------------------------
 TD-DOC-7-0001-...   |                  5 |                          2 |                     13
```

## Запрос 3 — безопасные IP без использования > 1 месяца

**Файл:** `query03_stale_safe_ip.sql`

**Логика.** `login_on_safe` — только активные safe IP + **LEFT JOIN** входов с этого IP. `last_other` — **DISTINCT ON** для последнего входа с другого IP. Фильтр: `IS NULL` или старше месяца.

**Пример (фрагмент):**

```
 IP-адрес    | Данные пользователя | Последний вход с IP | Дней без входа с IP | Заблокирован
-------------+---------------------+---------------------+---------------------+--------------
 10.200.x.x  | td_7_0001_...       | (пусто)             | (NULL)              | нет
```

## Запрос 4 — матрица активности (прошлая неделя)

**Файл:** `query04_activity_pivot.sql`

**Логика.**

- Прошлая полная календарная неделя: `[date_trunc('week', CURRENT_DATE) - 1 week, date_trunc('week', CURRENT_DATE))`.
- `eligible` — пользователи с хотя бы одним просмотром на этой неделе; `active` — слоты активности на той же неделе.
- Слоты: `slot_idx = hour / 3` (0…7), сетка `generate_series(0,6) × generate_series(0,7)`; колонки `пн_00-03ч` … `вс_21-24ч`.

Имена слотов: `пн_00_03`, `пн_03_06`, …, `вс_21_24`.

**Пример:** 19 пользователей (строк), широкая таблица; для просмотра удобно `\x` или выгрузка в CSV.

## Запрос 5 — отчёт по тематикам

**Файл:** `query05_themes_report.sql`

**Логика.** `theme_doc` → `doc_views` → `theme_stats` → `last_doc` (**DISTINCT ON** по `MAX(v.created_at)` в теме). Итоговый GROUP BY по теме.

**Пример (фрагмент):**

```
 Название тематики | Число документов | Общее число просмотров | Последний документ
-------------------+------------------+------------------------+-------------------
 TD-тема-7-0       |               12 |                     45 | TD-DOC-7-...
 Политика          |                1 |                      2 | Внутренняя переписка...
```
